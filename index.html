What actually goes wrong

Looked at 50 CLAUDE.md files across GitHub and personal projects. Same patterns kept appearing.

Most common - 73% of files

Duplicate rules

The same instruction written five different ways. "Never do X." "Always avoid X." "Do not, under any circumstances, X." The model reads all of it. Every call. You're paying tokens to say the same thing four extra times.

Common - 61% of files

Over-specified constraints

Rules sections that grew unchecked over months. Someone got burned once and added a rule. Then another. Now it's 1,200 tokens of "don't do this" and nothing about what to actually do. High cost, low value.

Frequent - 48% of files

Stale examples

Examples from an earlier version of the project that no longer match how things work. Still loaded every session. Costs tokens and sends the model toward outdated patterns.

Frequent - 44% of files

Missing tool scope

No explicit list of what tools exist or what they're for. The model guesses from context. Sometimes correctly. The files with clear tool definitions made noticeably fewer wrong assumptions.

Fixable - takes 20 minutes

Wrong structure for agent type

A solo dev setup trying to run a swarm. A minimal file trying to handle complex orchestration. The structure matters as much as the content. Different agent types need different CLAUDE.md shapes.
